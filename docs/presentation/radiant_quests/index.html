Radiant Quest System

Creating dynamic, procedurally generated quests in an Evennia-based MUD

Introduction

Radiant quests (a term popularized by Bethesda games like Skyrim) are procedurally generated, repeatable missions that provide variety and replayability to a game. Unlike static quests with fixed objectives, radiant quests dynamically select from a pool of possible locations, objectives, and rewards to create unique experiences each time.

In this tutorial, we'll build a complete radiant quest system for our Pythonic MUD, centered around a Boardwalk area with mini-games and randomized fetch quests. We'll cover everything from designing the data structures to implementing the gameplay mechanics, and we'll explore how these systems can be extended for more complex gameplay.

What We'll Build

Boardwalk Area

A central hub with NPCs, games, and attractions where players can spend tokens earned from quests.

Random Maze Generation

A procedurally generated maze that changes layout and item placement with each visit.

Quest Generator

A system that creates varied objectives based on item properties like color, size, and shape.

Token Economy

A reward system where players earn tokens to spend on Boardwalk games and attractions.

Prerequisites

To follow along with this tutorial, you'll need:

  • Working Evennia MUD installation (version 0.9.5+)
  • Basic understanding of Python and object-oriented programming
  • Familiarity with MUD concepts and Evennia's architecture
  • The character attributes system we implemented in the previous sprint
Note: All code in this tutorial is designed to integrate with the Pythonic MUD project structure. If you're using a different setup, you may need to adapt the import statements and class inheritance patterns.
